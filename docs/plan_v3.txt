Sinex Refactoring Plan (v3) — NATS‑Native Ingestion & Archival

Purpose: Eliminate the gap between envisioned architecture and the real code by making Events and Source Materials first‑class on NATS JetStream, with ingestd acting as a pure archiver. This plan is designed for a single‑pass refactor by one coding agent.

Canonical Decisions
- Protocol: NATS JetStream for all inter‑service communication (no gRPC in the hot path).
- Roles: Ingestors publish; ingestd archives; automata consume and publish.
- Streams:
  - events.raw: provisional Events published by satellites; consumed by ingestd; used by automata.
  - source_material.slices: byte slices of Source Material; consumed by ingestd assembler.
  - events.confirmations: ingestd emits “persisted” confirmations for provisional Events.
- Schema & DB:
  - Canonical migrations live in crate/lib/sinex-schema; apply via `just migrate` and update SQLx caches via `just sqlx-prepare`.
  - TimescaleDB required in dev (hypertable creation present in migrations).
- Event invariants: ULID IDs, rigorous ts_ingest/ts_orig, provenance is Material or Synthesis, JSON Schema mapping/versioning.
- Environment namespacing: Use SinexEnvironment for DB names, stream names, and subjects.

Data Model & Invariants
- Event (dynamic JSON payload):
  - event_id (ULID), source, event_type, payload, ts_orig, host, ingestor_version?, payload_schema_id?
  - Provenance:
    - Material: material_id (ULID), anchor_byte (i64), offset_start?, offset_end?
    - Synthesis: source_event_ids (NonEmpty), optional operation_id
- Idempotency:
  - Use `Nats-Msg-Id` header for dedupe on all published messages.
  - DB uniqueness: `core.events(event_id)` unique; `raw.source_material_registry(id)` unique.
  - Ack-after-commit/finalize to guarantee exactly-once effects for stateful operations.

JetStream Layout
- Streams (names environment-prefixed via SinexEnvironment):
  - events.raw: subjects `events.raw` (or `events.raw.*` if sharding by source/type is desired), storage=file, retention=7–30d, explicit ack.
  - source_material.slices: subjects `source_material.begin`, `source_material.slices.*`, `source_material.end`, storage=file, retention=7–30d, explicit ack.
  - events.confirmations: subject `events.confirmations`, storage=file, retention=7–30d, explicit ack.
- Consumers: ingestd creates durable consumers for `events.raw` and `source_material.*` with explicit ack and safe pull/stream settings.
- Payload limits: enforce slice size ≤ 512KB (configurable), below JetStream max payload.

Message Specifications
- `source_material.begin` (JSON, subject: `source_material.begin`)
  - `{ material_id: ULID, material_kind: string, source_identifier: string, metadata: JSON, started_at: ISO8601 }`
- `source_material.slices.<material_id>` (binary payload)
  - Headers:
    - `Nats-Msg-Id`: ULID (dedupe)
    - `Slice-Index`: u32
    - `Offset`: i64 (byte offset)
    - `Chunk-Hash`: blake3 hex
    - `Total-Slices?`: u32
    - `Content-Type?`: string (e.g., `text/plain; charset=UTF-8`)
- `source_material.end` (JSON, subject: `source_material.end`)
  - `{ material_id: ULID, ended_at: ISO8601, content_hashes: { blake3: string, sha256?: string }, total_slices: u32, total_size_bytes: u64 }`
- `events.raw` (JSON, subject: `events.raw`)
  - Event with required provenance (Material or Synthesis). Headers: `Nats-Msg-Id` = `event_id` ULID.
- `events.confirmations` (JSON, subject: `events.confirmations`)
  - `{ event_id: ULID, persisted: true, ts_ingest: ISO8601 }`

Ingestor Satellites (Publishers)
- Acquisition flow (SDK `AcquisitionContext`):
  - `start_stream(material_kind, source_identifier, metadata) -> handle { material_id }` publishes `source_material.begin`.
  - `append_slice(&[u8], headers{content_type?, chunk_hash}) -> (material_id, anchor_byte)` publishes `source_material.slices.<material_id>`.
  - `finalize()` publishes `source_material.end`.
  - `publish_event(Event)` publishes the event to `events.raw` (with material provenance using the returned `material_id, anchor_byte`).
- Notes:
  - Maintain in-memory rolling `anchor_byte` for each slice append; for text streams, anchor at content boundary (bytes).
  - Batch event publishing where possible; set `Nats-Msg-Id` from `event_id`.

ingestd (Archiver)
- Streams setup:
  - Ensure JetStream streams exist with correct config and subjects (namespaced via SinexEnvironment).
- Material consumer (assembler):
  - On `begin`: create ephemeral state (in memory or a small DB table); open temp file; initialize blake3/sha256 contexts.
  - On `slices.*`: append to temp file at offset; update hashes; track slice count and size; dedupe via `Nats-Msg-Id`.
  - On `end`: verify computed hashes; move temp file into annex/minimal blob store; insert/complete `raw.source_material_registry` with optional_blob_id, kind, source identifier, metadata, staged timestamps; commit and ack.
  - Timeouts: mark abandoned materials past threshold; reaper task cleans temp files; retries safe via idempotency.
- Event consumer:
  - Validate JSON schema; map to `core.events` fields; batch insert using UNNEST; enforce uniqueness by `event_id`.
  - Publish `events.confirmations` on success; ack after DB commit.
- Outbox:
  - Remove DB outbox publish-to-NATS for events (superseded by `events.raw`). If needed temporarily, publish `events.persisted` and deprecate post-migration.

SDK Changes
- `sinex-satellite-sdk`:
  - Add a `NatsPublisher` and `AcquisitionContext` (NATS-backed) with APIs: `start_stream()`, `append_slice()`, `finalize()`, `publish_event()`.
  - Provide default slice size (256–512KB). Use SinexEnvironment for subject namespacing.
- Optional feature flag: a `grpc-ingest` fallback, off by default.

Compatibility (Optional)
- If automata rely on persisted-event broadcasts, keep a temporary publish of `events.persisted` after DB commit in ingestd and migrate automata to consume from `events.raw` + `events.confirmations`.

Security & Observability
- Security:
  - Subject ACLs: satellites publish only to `source_material.*` and `events.raw`; ingestd consumes; automata consume `events.raw` and optionally `events.confirmations`.
- Metrics/logging:
  - ingestd: slices/sec, bytes/sec, finalize latency, hash mismatch count, DB batch sizes; event batch latencies, confirmation publish latencies.
  - satellites: slice sizes, total material bytes, per-event publish outcomes.
- Environment isolation: always prefix stream/subjects via SinexEnvironment (dev/staging/prod).

Single-Pass Implementation Order (Agent)
- JetStream setup (ingestd):
  - Add code to create/get streams `events.raw`, `source_material.slices`, `events.confirmations`.
- ingestd consumers:
  - Implement material assembler consumer (`begin`/`slices`/`end`) with temp files + hashes + DB registry finalize.
  - Implement event archiver consumer (validate → batch insert → confirmation publish).
  - Remove legacy outbox→NATS for events (or guard behind feature).
- SDK:
  - Add `AcquisitionContext` (NATS publisher) with the APIs above.
  - Wire satellites to use SDK for begin/slice/end and event publishing; ensure provenance includes material coords.
- Delete gRPC ingestion endpoints once end‑to‑end verified (no fallback retained).

Minimal Verifications
- Local (NATS + Postgres + annex/minimal store):
  - Publish a small text material (3 slices) + a provisional event; ingestd writes registry + events; confirmation arrives.
  - Re-run with deduplicated messages (same `Nats-Msg-Id`) and confirm idempotence.
  - Stop ingestd; publish 100MB worth of slices; restart ingestd; confirm archival completes from JetStream backlog.
- DB checks:
  - `core.events` rows with correct provenance for material-based and synthesis events.
  - `raw.source_material_registry` row complete with expected metadata and optional blob_id.

File & Code Pointers
- ingestd:
  - Streams + consumers: `crate/core/sinex-ingestd/src/service.rs` (add NATS consumers); `schema_sync.rs` unaffected.
- Remove gRPC paths in `main.rs`, `lib.rs` after verification (no fallback retained).
- SDK:
  - NATS publisher + `AcquisitionContext`: new module under `crate/lib/sinex-satellite-sdk/src/` (e.g., `publisher.rs` + `acquisition.rs`).
  - Update example processors to show begin/slice/end + events.raw.
- Satellites:
  - FS/System/Desktop/Terminal: replace current ingestion path with SDK calls (`crate/satellites/*/unified_processor.rs`).
- Schema/DB:
  - Keep `sinex-schema` as canonical; consider small helper table only if assembler needs persistent state beyond temp files.

Acceptance Criteria
- ingestd archiver runs with two JetStream consumers (materials + events) and builds cleanly.
- SDK satellites publish to NATS; no gRPC in hot path.
- End‑to‑end capture: begin→slices→end → registry finalized → event persisted → confirmation emitted.
- Idempotent retries: no duplicate rows on replays; dedupe via `Nats-Msg-Id` and DB uniques.

Notes From Current Code Reality
- TimescaleDB: migrations create hypertable; treat as required in dev.
- Migrations: live in `crate/lib/sinex-schema`; do not introduce a new migrations crate.
- Environment namespacing: `SinexEnvironment` already provides DB/subject names; reuse for JetStream subjects.
- Event builder requires provenance: SDK `append_slice` must return `(material_id, anchor_byte)` to build material provenance events.
- Gateway/CLI are orthogonal to ingestion; keep CLI default to TCP in dev (`127.0.0.1:9999`); not in ingestion scope.

Completeness Checks & Operational Details
- Supersedes: This plan supersedes `docs/plan_v2.txt` and the ingestion parts of `docs/TARGET_final.md`.
- Schema registry handling:
  - Ingestd loads active schemas from `sinex_schemas.event_payload_schemas` at startup and refreshes on a timer (e.g., 60s) or upon admin signal.
  - Schema selection: key on (source, event_type), apply latest active version unless an explicit `payload_schema_id` is provided in the Event.
- Source material registry persistence:
  - Table: `raw.source_material_registry` (id ULID, material_kind TEXT, source_identifier TEXT, status TEXT, timing_info_type TEXT, metadata JSONB, staged_at TIMESTAMPTZ, start_time?, end_time?, staged_by?, staged_on_host?, optional_blob_id ULID?).
  - On `begin`: insert with status `sensing`, timing_info_type `realtime` unless specified; set start_time.
  - On `end`: set status `completed`, end_time, optional_blob_id, and finalize metadata.
- Assembler state & restart semantics:
  - For resilience across restarts, maintain a lightweight `assembler_state` record (material_id, temp_path, next_offset, slices_seen, started_at, last_update_at). Cleanup if idle > threshold (e.g., 30m).
  - On restart, resubscribe and resume assembly by reading from JetStream starting at the last acked sequence; dedupe by `Nats-Msg-Id`.
  - Slice ordering: apply by `Slice-Index` and/or `Offset`; allow out‑of‑order arrival; verify continuity on finalize.
- NATS consumer settings (guidance):
  - Explicit ack, AckWait ≥ 30s, MaxAckPending sized to expected throughput, Pull batch size tuned (e.g., 256), durable name namespaced per environment.
  - Use `headers_only` filters where applicable to short‑circuit malformed messages early.
- Hashing & integrity:
  - Maintain per‑slice blake3 verification (header `Chunk-Hash`) and rolling file hash; verify totals against `source_material.end`.
  - On mismatch, mark material `failed` and emit a diagnostic event; do not ack the end message until handled.
- Annex/minimal blob store:
  - Start with a minimal filesystem store if annex isn’t configured; the interface should abstract write/move/verify so moving to annex is a swap.
- Removal of legacy paths:
  - Once satellites publish to JetStream and ingestd consumers are verified, delete gRPC ingest paths and DB outbox→NATS event publishing code.
- Environment/config:
  - JetStream subjects and stream names must use `SinexEnvironment` prefixes (dev./staging./prod.).
  - Config via env: `SINEX_NATS_URL`, stream names, slice size, ack settings; reasonable defaults baked in.
- Testing notes:
  - Property tests around idempotence (replay same messages), out‑of‑order slices, partial material then timeout, and multi‑GB materials across restart.
  - Integration tests for schema loading/refresh and event validation fallbacks.
  - DB invariants: enforce `CHECK` for provenance XOR (Material xor Synthesis) and `UNIQUE` on `(material_id, anchor_byte)` for material events.
  - Serializer compatibility: where sources evolve (e.g., sysinfo), consider `#[serde(untagged)]` strategies to deserialize historical Source Material during replays.
