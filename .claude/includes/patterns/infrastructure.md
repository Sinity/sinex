## Prelude Imports (USE for common imports)

```rust
// Recommended: flattened imports for most code
use sinex_primitives::prelude::*; // Event, Id, SinexError, validation, domain types
use sinex_db::DbPoolExt;          // Repository access (no prelude module — import directly)

// Test utilities (via xtask sandbox feature in tests)
#[cfg(test)]
use xtask::sandbox::prelude::*;   // TestContext, sinex_test, Timeouts, etc.
```

---

## Test Policy (Attribute + Placement)

- Default to `#[sinex_test]`.
- Raw `#[test]` / `#[tokio::test]` is allowlisted only for `trybuild` and proc-macro-internal tests.
- Prefer per-crate `tests/` files by default.
- Inline `#[cfg(test)]` is exception-only for small internal tests where `tests/` extraction would require undesirable visibility changes.
- Every raw-test or inline-test exception must include a short in-file reason.

## Event Creation

**Typed Payloads (preferred):** Use fluent methods on `EventPayload` types.

```rust
use sinex_primitives::prelude::*;  // EventPayload, typed payloads re-exported

// Material provenance (ingestors - raw source data)
let event = FileCreatedPayload { path: "/file.txt".into(), size: 1024, ... }
    .from_material(source_material_id)
    .build()?;

// Synthesis provenance (automatons - derived from other events)
let event = AnalyticsSummaryPayload { ... }
    .from_parents(parent_event_ids)?
    .build()?;

// Direct construction with explicit provenance
let event = payload.into_event(provenance);
```

**Dynamic Events (escape hatch):** For JSON payloads without typed structs.

```rust
use sinex_primitives::prelude::*;  // EventBuilder, DynamicPayload re-exported

// Use DynamicPayload for runtime source/type
let payload = DynamicPayload::new("source", "event.type", json!({...}));

// Or EventBuilder::dynamic() for full provenance chain
let event = EventBuilder::dynamic("source", "event.type", json!({...}))
    .from_material(material_id, anchor_byte)
    .build()?;
```

## Error Handling | ALWAYS USE SinexError

```rust
use sinex_primitives::prelude::*;  // SinexError, Result re-exported

// Create errors with context
SinexError::validation("Invalid input")
    .with_context("field", "username")
    .with_context("reason", "too short")

// Add explicit context fields when propagating failures
fn process_data(input: &str) -> Result<Output> {
    do_work(input).map_err(|e| {
        SinexError::processing("failed to process data")
            .with_context("input_len", input.len().to_string())
            .with_std_error(&e)
    })
}
```

## Privacy Engine (USE instead of manual redaction)

```rust
use sinex_primitives::privacy::{self, ProcessingContext};

// Process text through the unified privacy engine
let result = privacy::engine().process("export TOKEN=ghp_abc123", ProcessingContext::Command);
if result.any_matched() {
    println!("Redacted: {}", result.text);  // Cow<str> — zero-alloc if unchanged
}
if result.suppressed {
    // Drop the containing field entirely
}

// Available contexts: Command, Clipboard, WindowTitle, Journal, Dbus, Notification, Document, Metadata
// Available strategies: Redact, Encrypt (XChaCha20-Poly1305), Hash (BLAKE3 MAC), Suppress
```

**Key types:** `PrivacyEngine`, `PrivacyConfig`, `ProcessingContext`, `Strategy`, `PatternRule`, `Processed<'a>`

**Architecture:** Single global engine via `privacy::engine()` (initialized from `PrivacyConfig::from_env()`).
Replaces the old `secret_redaction` + `redaction_config` modules. All sensitive-data handling flows through here.

## Domain Enums (USE instead of stringly-typed fields)

These typed enums replace raw `String` fields throughout the RPC and audit APIs:

```rust
use sinex_primitives::domain::*;

// Operational state tracking
let status: OperationStatus = OperationStatus::Running;  // Running, Success, Failed, Cancelled, Pending
let health: HealthStatus = HealthStatus::Healthy;        // Healthy, Degraded, Unhealthy, Bypassed
let tier: DataTier = DataTier::Live;                     // Live, Archive, Tombstone
let outcome: ReplayOutcome = ReplayOutcome::Success;     // Success, Failed, Cancelled
let blob_status: BlobVerificationStatus = BlobVerificationStatus::Verified;  // Pending, Verified, Corrupted
let node_type: NodeType = NodeType::Ingestor;            // Ingestor, Automaton, Service, Processor
let node_state: NodeState = NodeState::Running;          // Running, Draining, Paused, Failed, Unknown
```

Event payload field enums (in `sinex_primitives::events::enums`):

| Category | Enums |

| Filesystem | `FileModificationType` |
| Process | `ShutdownReason`, `DeactivationReason` |

| Power/HW | `PowerEventType`, `BluetoothEventType`, `DeviceEventType`, `DeviceType` |
